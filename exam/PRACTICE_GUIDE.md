# 시험 제출 시스템 연습 안내

## 개요

중간고사/기말고사는 **GitHub 기반 자동 채점 시스템**으로 진행됩니다.
테스트용 5문항으로 **답안 작성 → push → 포맷 검증 → 수정** 흐름을 미리 연습하세요.

> 이번은 **연습**입니다. 점수에 반영되지 않습니다.

---

## 연습 방법

### Step 1. 본인 레포에서 exam 폴더 확인

본인의 `3-<GitHub ID>` 레포에 아래 파일이 배포되어 있습니다.

```
exam/
├── README.md         ← 응시 안내 (형식 규칙 설명)
├── questions.md      ← 테스트 문제 5문항
└── answers.md        ← 답안 파일 (이것만 수정!)
```

### Step 2. 답안 작성

`exam/answers.md`를 열고 각 문항의 `답:` 뒤에 답을 기재하세요.

```markdown
## Q1
답: 3

## Q2
답: 2

## Q3
답: 1, 3

## Q4
답: T

## Q5
답: 2.60
```

**형식 요약:**

| 유형 | 형식 | 예시 |
|------|------|------|
| 단일선택 | 숫자 1개 | `답: 3` |
| 복수선택 | 쉼표+공백 구분 | `답: 1, 3, 5` |
| 참/거짓 | T 또는 F | `답: T` |
| 숫자 | 소수점 포함 | `답: 2.60` |

### Step 3. main 브랜치에 push

```bash
git add exam/answers.md
git commit -m "답안 제출 연습"
git push origin main
```

> **`main` 브랜치만 인식됩니다.** 다른 브랜치에 push하면 검사 대상에서 제외됩니다.

### Step 4. Issues 탭에서 포맷 검증 확인

30분마다 자동 검사가 실행됩니다. 본인 레포의 **Issues** 탭을 확인하세요.

- **📋 답안 포맷 상태** Issue가 자동으로 생성/업데이트됩니다.
- ✅ `모든 문항 파싱 성공` → 제출 형식 정상
- ❌ 오류 표시 → 해당 문항 형식을 수정하고 다시 push

### Step 5. 오류가 있으면 수정 후 재push

형식을 고치고 다시 push하면 다음 검사 때 Issue가 자동 업데이트됩니다.

---

## 주의사항

- **채점 결과(점수)는 공개되지 않습니다.** 포맷이 맞는지만 확인할 수 있습니다.
- `exam/questions.md`와 `exam/README.md`는 수정하지 마세요.
- 실제 시험 때는 문제가 교체되고 마감 시각이 설정됩니다.
